# FEATURE-001: 支持创建任务

## 背景

示例项目需要一个足够小但完整的业务功能，用来验证 Issue、设计文档、代码、测试、PR、Actions 和发布记录之间的闭环。

## 目标

提供最小任务 API：

- 创建任务。
- 查询任务列表。
- 更新任务状态。

## 非目标

- 不做用户登录。
- 不做多人协作。
- 不引入数据库。
- 不引入前端页面。

## 技术方案

- 使用 Node.js 内置 `http` 模块提供 API。
- 使用 JSON 文件做本地持久化。
- 使用 Node.js 内置 `node:test` 做自动化测试。
- 不引入外部 npm 依赖，降低 bootstrap 成本。

## API

| 方法 | 路径 | 说明 |
|---|---|---|
| `GET` | `/health` | 健康检查 |
| `GET` | `/tasks` | 查询任务列表 |
| `POST` | `/tasks` | 创建任务 |
| `PATCH` | `/tasks/:id/status` | 更新任务状态 |

## 风险

- JSON 文件存储不适合并发写入和生产环境。
- 当前 API 没有鉴权。
- 当前状态模型只覆盖 demo 场景。
