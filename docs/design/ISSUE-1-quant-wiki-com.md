# ISSUE-1: 提取 https://quant-wiki.com/ 文章中所有的量化方案

## 来源

- Issue: https://github.com/givingwu/ai-workflow-demo/issues/1
- AI 可执行等级：auto-pr

## 背景

因为目前的量化市场非常的 technical，需要提炼很多策略，用到很多机器学习的算法等。

## 目标

找到一个能够赚钱的策略

## 非目标

不要尝试进行很复杂的超越了当前设备运行能力的计算

## 影响范围

整个工程

## 验收标准

- [ ] 能获取数据
- [ ] 能结构化数据
- [ ] 能用数据调试多种策略
- [ ] 能懂动态切换不同策略
- [ ] 能回测数据
- [ ] 能在线模拟盘测试数据

## 初始执行方案

1. 读取 Issue 中的背景、目标、非目标和验收标准。
2. 先补充设计文档，等待人工 review。
3. 设计 PR 合并后，再进入实现分支。
4. 实现阶段必须补充测试，并通过 CI 后再请求 review。

## 风险与人工 Gate

- 该 PR 只沉淀设计，不直接修改业务代码。
- 如果需求涉及外部网站、生产数据、密钥、付费 API 或高风险操作，必须再次等待人工确认。
- 任何发布动作都必须通过独立的发布确认 Gate。
